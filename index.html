Introducción a Git

Git es una herramienta para el control de versiones de un proyecto durante su desarrollo. Generalmente se encuentra asociado con el desarrollo de productos de software. Es importante mencionar que Git y Github son herramientas diferentes, la última es una herramienta colaborativa en Internet que usa Git internamente para gestionar los proyectos.

Los comandos principales para trabajar correctamente con GIT son el: git clone, git init, git add, git commit, git push origin rama, git pull origin rama, git branch rama, git checkout rama, git status, git log --oneline, git merge, git commit -am "mensaje", etc

Las buenas prácticas indican que siempre se debe hacer un pull del repositorio remoto, y luego hacer los push. De igual forma, es importante estar parados en la rama correcta antes de hacer el pull, ya que en el transfondo hace un merge.

Cherry pick es un comando realmente util, para seleccionar un commit o un conjunto de ellos y pasalos/aplicarlos a otra rama. No es una buena práctica pero yo le encuentro utilidad en este tipo de escenarios (para pasar a la rama correcta, commits que fueron confirmados accidentalmente en otra rama)
